# 蛋黄桌宠产品需求

## 产品定位

蛋黄桌宠从“个人纪念桌宠”升级为“宠物陪伴平台”。纪念宠物是核心情感板块之一，但产品不只服务纪念场景，也支持在世宠物、虚拟宠物、多宠物家庭、AI 陪伴、提醒、故事和商业化扩展。

## 目标用户

- 有真实宠物照片，希望把宠物放到桌面长期陪伴的人。
- 有纪念宠物需求，希望用克制、温和方式保存照片、故事和日常陪伴的人。
- 喜欢桌面小组件、轻量提醒、AI 角色陪聊的人。
- 多宠物家庭，希望每只宠物有独立档案、动作、等级和记忆的人。

## MVP 范围

必须保留：

- 单机可用。
- 默认宠物和多宠物切换。
- 基础动作、拖动、右键菜单、气泡和低频自动交流。
- AI Provider 可配置，但 Key 不进公开包。
- 本地待办和提醒。
- 宠物导入和基础动作资产管理。
- 可发给陌生用户安装的 Windows 包。

暂缓：

- 社区和素材市场。
- 复杂云同步。
- 移动端实时联动。
- 多人家庭账号。
- 自动公开发布和素材二次分发。

## 产品模式

### 宠物陪伴

- 桌面巡游、边缘安静、当前屏自由、多屏巡游。
- 摸摸头、召回、说句话、安静模式。
- 气泡不抢焦点，不挡宠物脸。

### 宠物创建

- 上传真实宠物照片。
- 宠物类型必须有细分类，不只猫狗；至少覆盖犬类、猫类、兔类、小型哺乳、大型哺乳、鸟类、爬行动物、水生生物、昆虫/节肢、人物、幻想生物、机器人、拟物/物件和其他。
- 生成主形象和基础动作。
- 不同分类需要有不同的默认动作语义，例如人物是站立/走路/招手，鸟类是跳步/拍翅，水生生物是漂浮/游动，拟物角色是轻晃/弹跳。
- 后续扩展动作包必须经过身份、比例、基线和透明边缘 QA。

### 纪念空间

- 保存故事、照片、日期和回忆。
- 表达克制，不宣传“复活宠物”。
- 默认本地优先，上传前二次确认。

### 日常宠物

- 在世宠物档案。
- 日常语气、成长记录、趣味动作。
- 可与提醒和待办联动。

### AI 陪聊

- 宠物人格短句化。
- 结合长期记忆、近期对话和本地待办。
- 支持本地时间直答；用户问当前时间、日期、星期时不依赖云端模型。
- 支持本地待办查询；用户问今天安排、未完成、逾期或重要事项时，不依赖云端模型也能读取本机待办摘要。
- 支持明确触发的联网资料查询和通用知识问题查询；AI 可用时基于检索资料组织回复，AI 不可用时给本地网页摘要兜底。
- 搜索能力需要至少具备主搜索源和知识类备用来源，单一网页解析失败时不能直接让问答链路失效。
- 聊天界面需要展示云端模型、时间直答、资料查询、待办摘要和本地兜底的当前状态，避免用户误以为所有回复都来自同一通道。
- 聊天快捷入口需要可换行或均分排列，时间、待办、查资料和背景等工具入口不能因窗口变窄而溢出。
- 聊天支持可选“角色 Skill”，在保留当前宠物身份的前提下切换技术导师、产品拆解、研究助手、直说教练、运营写手等表达风格；默认不冒充具体真人达人。
- 不做客服、不讲大道理、不替代医疗或心理治疗。

### 商业模块

- 免费版：默认宠物、基础动作、本地提醒、少量 AI 对话、本地数据。
- 付费版：真实宠物生成、多宠物家庭、高级动作包、云同步、更多 AI 额度、纪念空间。
- 增值项：节日动作包、主题皮肤、气泡主题、桌面小组件。

## 非功能需求

- 低资源占用，长期挂桌面不打扰。
- 默认浅色暖色风格，适合日常使用。
- 错误给明确反馈，不能让用户看到 Python 堆栈。
- 所有用户数据可导出、可删除。
- 安装包不要求用户安装 Python。
