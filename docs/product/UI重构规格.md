# UI 重构规格

当前 Tk 版本 UI 优化的执行级清单见 `docs/product/当前Tk桌宠UI优化清单.md`。面向外部设计工具的总体说明见 `docs/product/蛋黄桌宠总体UI设计说明.md`。本文件保留总体设计规格；具体 Tk 迭代、截图验收和 exe 打包要求以该清单为准。

## 视觉方向

默认方向是“温暖、干净、可长期挂桌面的陪伴产品”，不是科技后台。

当前工程进展：

- Tk 右键菜单、气泡和窗口行为已有独立 UI 状态模型。
- E 盘 Tk 运行镜像已开始用右键菜单模型驱动基础动作、扩展动作和更多动作边界。
- 当前主线已收敛为 Python/Tk；WPF Spike/技术验证目录已移除，不再作为当前 UI 同步目标。
- 每次完成 Tk UI 优化后，都要同步 E 盘运行镜像并重新生成 Windows exe 包。

颜色：

- 背景：暖白、浅米色。
- 主色：蜂蜜色。
- 状态：少量鼠尾草绿。
- 危险：柔和红。
- 避免：大面积深色、厚重边框、复杂渐变、营销卡片。

## 信息架构

控制面板建议固定或准固定尺寸，左侧导航、右侧内容，正文区域滚动。

页面：

- 首页：当前宠物、状态、今日陪伴、AI 状态、待办、快捷操作。
- 宠物：宠物列表、档案、照片、主形象、动作包、切换。
- 陪伴：等级、互动、说话频率、安静时段、自动交流。
- 行为：大小、速度、活动范围、多屏策略、拖动手感。
- 对话：气泡样式、颜色、聊天背景、AI Provider、记忆。
- 提醒：待办列表、重复、稍后提醒、时间轴。
- 生成：新宠物导入、动作生成、QA、失败重试。
- 安全：数据目录、导出、备份、隐私、清理、日志。

## 控件规则

- 高频按钮点击区域接近 44px。
- 每页只有一个主动作。
- 下拉、滑杆、开关、列表统一样式。
- 错误有明确反馈，不展示堆栈。
- 气泡不抢焦点，不挡宠物脸。
- 右键菜单短、稳、分组明确。
- 当前宠物在列表中排第一，状态明确显示“当前”。
- 宠物类型使用细分类 chip，不再只有自由文本种类；编辑和新增时必须展示分类说明和默认动作语义。
- 列表操作按钮不得放进会被压缩成竖排的窄列；常用操作应进入卡片内容区、工具栏或可换行操作行，并使用稳定宽度。

## 宠物交互

- 左键拖动。
- 双击摸摸头。
- 右键快捷菜单。
- 悬停轻微反馈。
- 拖动时四肢跑动，停止后自然回到待机。
- 活动三档：边缘安静、当前屏自由、多屏巡游。
- 基础动作槽位保持兼容，但动作语义按分类变化：人物可走路/招手，鸟类可跳步/拍翅，水生生物可漂浮/游动，幻想/拟物角色可弹跳或漂浮。

## 交流策略

- 自动说话低频、可关闭、支持安静时段。
- 主动点击“说句话”时更亲近。
- AI 回复默认短句，不做客服、不讲大道理。
- 当前时间、日期、星期优先本地直答；联网查询必须由“查一下/搜索/最新/资料”等明确意图触发，并显示查询状态。
- 什么是、谁是、为什么、怎么、如何、哪里、哪年、有哪些等通用知识问题也应触发资料查询；搜索源应有备用来源，不能只依赖单个页面解析。
- 普通知识问句应先做实体词清洗，再尝试百科摘要和网页搜索合并；实时、新闻、价格、天气类问题不应被百科摘要覆盖。
- 无 AI 时的资料回复必须显示来源类型和 URL，不能只给无出处摘要。
- 今天安排、待办列表、未完成、逾期和重要事项优先读取本地待办摘要；“查看待办”和“新增提醒”必须明确区分。
- 聊天窗必须显式展示云端、时间、资料、待办和本地兜底能力状态，让用户知道当前会用 AI、本机工具还是本地摘要。
- 聊天窗快捷工具区必须支持换行或均分网格，时间、待办、资料和背景等入口不能依赖单行横向堆叠。
- 聊天窗需要提供角色 Skill 选择入口；角色只改变表达风格，不能覆盖宠物身份、主人关系和真实故事边界。
- 角色 Skill 必须展示当前角色的适用场景和边界说明；泛化达人风格只能借表达方法，不模仿具体真人、口头禅、个人经历或具体平台人设。
- AI 未配置或失败时，资料查询可降级为本地网页摘要；不能假装已生成完整 AI 结论。
- 纪念模式更克制、更安静。
- 日常宠物模式更活泼。

## 验收标准

- 文本不溢出、不重叠。
- 按钮文字、状态 chip、长标题和长说明不竖排、不重叠、不横向溢出。
- 小窗高度下仍能找到低频页面。
- 页面切换后滚动位置回到顶部。
- 下拉和弹层不会被裁切。
- 右键菜单内部按钮不会被失焦提前关闭。
- 大 UI 改动必须留截图或验收记录。
