## Review Notes

- `build_pet_switcher_model()` 同时接受运行期 `id` 和资产 manifest `pet_id`，避免 `pet-family.json` 与 `validate_phase3.py` 使用不同字段名导致模型丢项。
- Tk 弹层只按 `pet_id` 映射回本地 pet 数据生成照片，纯 model 不携带图片对象、源图路径或隐私资源。
- 当前宠物排序、`selected` variant、隐藏数量和管理入口已有单元测试覆盖。
- `validate_phase3.py` 已输出 `pet_switcher` 摘要，便于后续设计工具读取当前模型结构。
- 弹层关闭、管理入口、切换宠物命令仍沿用原本 Tk 延迟执行路径，降低右键面板提前销毁导致入口失效的风险。

## Conclusion

Approved。下一步建议继续把弹层定位策略、行样式 token 和面板尺寸约束抽到 view model，减少单文件中散落的 Tk 专用状态。
