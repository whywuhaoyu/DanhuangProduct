# 产品路线图

## Phase 0：资产冻结和产品定义

状态：进行中，核心拆分已进入 UI 状态模型和单文件接入阶段。

- 已完成当前原型完整归档。
- 已生成文件清单、隐私清单、宠物家庭清单、代码结构清单。
- 已建立产品化工程根目录。
- 下一步补充竞品/用户画像、MVP 验收口径、商业模式细化。

## Phase 1：Python/Tk 原型产品化

目标：让现有原型稳定、可验证、可拆分，不继续把功能堆进单类。

任务：

- 保持当前运行版稳定。
- 修复错误日志中的有效新异常；旧异常只记录，不重复修。
- 保持 `tk.Menubutton + tk.Menu` 为稳定下拉兜底。
- 模块化拆分 `core/*` 和 `ui/*`，每步保持行为不变。
- 建立当前 Tk UI 优化清单，把可见体验改动固化为 Tk 产品资产。
- 重做控制面板信息架构和截图验收。
- 建立原型导出清单和隐私清理流程。
- 每次完成 Tk 优化后，同步 E 盘运行镜像并重新生成 Windows exe 包。

验收：

- `python -m py_compile run-danhuang-desktop-pet.py` 通过。
- settings、family、state、todos、providers JSON 可解析。
- 右键、控制面板、气泡、拖动、多屏、提醒、AI 测试可人工验收。

## Phase 2：Tk Windows 可分发版

目标：把当前 Tk 运行版稳定成可持续分发的 Windows exe。

任务：

- 继续拆分单文件运行入口，但每一步保持 E 盘运行镜像可启动。
- 建立固定打包流水线：源码同步、隐私重置、PyInstaller one-folder、exe 烟测、zip 和 SHA256。
- 补齐控制面板、右键菜单、聊天窗、提醒页、AI 页和动作页的截图验收。
- 保持安装包不携带聊天、待办、提醒历史、API Key、DPAPI blob、本机路径和日志。

当前状态：

- WPF Spike/技术验证目录已按 2026-07-03 决策移除。
- Tauri/Vue 只保留为历史实验分支，当前优化不再围绕它补 parity。
- Tk `0.11.43` 是当前运行和打包基线。

## Phase 3：Windows 商业 MVP

目标：做出可发给陌生用户安装的版本。

任务：

- 迁移动作状态机和宠物窗口。
- 迁移控制面板。
- 迁移 AI、提醒、故事、导出。
- 接入自动更新、授权、崩溃日志和反馈入口。

## Phase 4：商业化基础

目标：从工具变成可运营产品。

任务：

- 账号、授权、订阅。
- 宠物生成额度。
- 云同步。
- 主题和动作包。
- 隐私协议、服务协议、支付和退款策略。

## Phase 5：macOS 与移动端

目标：跨平台共享资产和账号，不强行复用 UI。

任务：

- macOS 使用 SwiftUI/AppKit 做原生菜单栏和透明桌宠窗口。
- 移动端定位为陪伴管理 App：宠物档案、AI 聊天、照片/故事、生成任务、桌面端同步、Widget/Live Activity。
