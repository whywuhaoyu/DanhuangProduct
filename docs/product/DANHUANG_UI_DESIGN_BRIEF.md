# 蛋黄桌宠总体 UI 设计说明

更新时间：2026-06-18

本文用于交给其他设计工具生成总体 UI 方案。设计工具应把蛋黄理解为 Windows 桌面陪伴产品，不是网页、SaaS 后台、移动 App 或营销 landing page。

## 一句话定位

蛋黄桌宠是一个本地优先的 Windows 桌面宠物陪伴产品：宠物长期挂在桌面上，用透明小窗口、短句气泡、右键快捷面板、控制面板、AI 陪聊、本地提醒、多宠物档案和安全导出组成完整体验。

核心感受是“温暖、干净、安静、可长期陪伴”，不是“效率工具”“客服助手”或“科技后台”。

## 设计输出目标

设计工具至少生成这些界面：

1. 桌面宠物透明窗口：宠物精灵、气泡、低干扰陪伴状态。
2. 右键快捷面板：当前宠物、常用入口、基础动作、扩展动作、窗口操作。
3. 控制面板 Shell：原生标题栏、左侧分组导航、右侧滚动内容、底部保存/状态反馈。
4. 首页总览：当前宠物、陪伴等级、AI 能力、提醒摘要、快捷入口。
5. 形象页：当前主形象、现实照片、家人形象列表、新宠物导入入口。
6. 对话/聊天窗：能力状态条、角色 Skill、快捷工具区、消息列表、固定底部输入。
7. AI Provider 页：厂商状态卡、模型和连接配置、本机 Key、测试结果。
8. 提醒页：快速新增、筛选、待办卡片、详情编辑、到点弹窗、时间轴。
9. 动作页：动作预览、右键动作配置、扩展动作上传和 QA 结果。
10. 档案/故事页：宠物级档案、长期记忆、故事列表、全文阅读器。
11. 外观/行为/巡逻页：窗口行为、气泡样式、颜色预设、移动和多屏策略。
12. 安全页：个人备份、公开分发边界、安装包导出、隐私排除清单。

## 当前功能快照

当前 E 盘运行镜像包含：

- 5 个 ready 宠物：蛋黄、小墨、橘宝、小白、胖久。
- 动作 manifest 当前可识别 54 个可播放动作。
- 扩展动作条 11 个，包括橘宝的舔爪/舔毛/洗脸，以及胖久的摸摸头/舔爪/打哈欠/吐舌头/伸懒腰/打盹/追蝴蝶/闻一闻。
- AI Provider 预设 11 个，真实 Key 不展示、不导出。
- 本地待办、提醒历史、聊天记忆、长期记忆和故事都按本机数据读取。
- 当前主宠物是胖久，但产品 UI 不能写死某一只宠物；所有状态都要跟随当前宠物刷新。
- Tk 模块化已抽出右键菜单、气泡和窗口行为的 UI 状态模型；当前 E 盘单文件运行镜像已使用右键菜单模型驱动标题、副标题、常用入口、动作区、更多动作 footer、窗口命令、布局时序、主面板尺寸约束、主面板避让定位和按钮/弹层样式 token，并开始使用气泡模型托管样式/颜色/宽度/时长/定位、气泡 renderer 托管图片绘制、窗口模型托管基础尺寸。
- 当前主线已收敛为 Python/Tk；WPF 技术验证目录已移除，不再作为设计承接目标。
- 每次 Tk UI 优化完成后，设计和工程验收都需要落到 E 盘运行镜像、QA 截图和新的 Windows exe 包。

## 当前重构边界

设计工具输出时请把“当前能设计的目标 UI”和“当前已接入的工程边界”分清：

- 右键快捷面板的结构已经稳定为 `常用 / 基础动作 / 扩展动作 / 窗口` 四组，扩展动作超过 4 个进入“更多动作 +N”；当前 Tk 运行层已优先按 model item 的 `command/page/variant/enabled/action_id/loops` 渲染，“切换形象”弹层也已优先按 pet switcher model 渲染，并开始消费 right-menu style tokens、主面板宽度约束和主面板避让定位模型。
- 气泡模型已固定样式、颜色、尺寸、图片绘制和屏幕内位置规则；当前 Tk 运行层已消费样式、文字颜色、文本宽度、显示时长、屏幕内定位和 modular 图片 renderer，但逐字显示和外观页实时预览仍在旧单文件逻辑中。
- 桌宠窗口模型已固定大小、透明度、置顶、多屏巡逻等状态字段；当前 Tk 运行层已消费基础窗口尺寸并暴露窗口 view model，但透明窗口属性、拖动、多屏巡逻和真实图片渲染尺寸仍在旧单文件逻辑中。
- Tauri/Vue 仅保留为历史实验分支，当前设计稿不要以它或 WPF 为默认落地对象。

## 可直接用于 UI 生成的模型字段

### 右键快捷面板

来源：`src-prototype/modular/ui/tk_right_menu.py`

- `header`：当前宠物名称、状态、动作包摘要、是否当前宠物。
- `sections`：固定分组为 `common`、`base_actions`、`extension_actions`、`window`。
- `sections[].items`：每个按钮统一使用 `command`、`page`、`variant`、`enabled` 描述渲染和行为。
- `sections[].footer`：扩展动作区底部操作，例如“更多动作 +N”和“管理动作”。
- `base_actions`：基础动作，始终保留在右键面板。
- `visible_extension_actions`：默认可见扩展动作，当前上限为 4 个。
- `hidden_extension_actions`：进入“更多动作 +N”的扩展动作。
- `hidden_extension_buttons`：更多动作弹层直接渲染的动作按钮，包含 `action_id`、`label`、`enabled`、`loops`。
- `sections["window"].items`：隐藏气泡、退出等窗口命令；后续可继续扩展置顶、透明度或托盘相关命令。
- `layout`：右键面板列数、宽度建议、按钮最小高度、面板与宠物间距、屏幕边距、宠物避让边距、外部点击检测延迟和自动关闭时长。
- `style` / `right_menu_style_tokens()`：右键主面板、按钮 variant、弹层 shell、选中行、当前状态和 footer 按钮的颜色 token。设计工具应把这些视作当前 Tk 初版状态 token，而不是最终品牌色板。
- `compute_menu_position()`：根据宠物矩形、面板尺寸、屏幕矩形和点击点输出主面板 `x`、`y`、`placement`；默认右侧优先，右侧不可用时尝试左侧、下方、上方，再落到点击点附近，并尽量避开宠物窗口。
- `right_menu_popup_layout()`：右键子弹层间距、屏幕边距、关闭延迟和更多动作最大可见数量。
- `compute_popup_position()`：根据 anchor、右键面板、弹层尺寸和屏幕矩形输出 `x`、`y`、`placement`；默认右侧优先，右侧空间不足时翻到左侧，并夹在屏幕内。

### 切换形象弹层

来源：`src-prototype/modular/ui/tk_right_menu.py`

- `title`：弹层标题，当前为“选择形象”。
- `items`：可见宠物项，包含 `pet_id`、`label`、`current`、`enabled`、`variant`、`species`、`status`、`action_pack_level`。
- `hidden_count`：超过展示上限后被隐藏的宠物数量，用于提示“还有 N 个形象”。
- `empty`：没有可切换形象时的空态标题和说明。
- `footer`：底部管理入口，当前指向控制面板“形象”页。
- `layout`：最多展示宠物数、行高和缩略图尺寸。
- `popup_layout`：弹层间距、屏幕边距、关闭延迟和更多动作可见数量；同一套规则也用于“更多动作”弹层。

照片不进入 model。Tk 渲染层应按 `pet_id` 去对应资源目录取当前主像素图或待机帧缩略图。

### 气泡

来源：`src-prototype/modular/ui/tk_bubble.py`、`src-prototype/modular/ui/tk_bubble_render.py`

- `style`：`soft`、`rounded`、`cloud`、`note`、`caption`。
- `colors`：`fill`、`outline`、`text`。
- `shadow`：基于填充色计算的柔和阴影色。
- `duration_seconds`：2-12 秒范围内的显示时长。
- `size`：`body_width`、`body_height`、`canvas_width`、`canvas_height`、`tail_y`。
- `text_width`：Canvas 文本换行宽度。
- `compute_bubble_position()`：根据宠物矩形、气泡尺寸和屏幕矩形输出 `x`、`y`、`placement`。
- `render_bubble_image()`：根据样式、尺寸、尾巴位置和颜色输出 RGBA 气泡图片；支持 `soft / rounded / cloud / thought / note / caption`。

### 桌宠窗口

来源：`src-prototype/modular/ui/tk_pet_window.py`

- `size`：`scale`、`width`、`height`，当前只代表基础窗口尺寸。
- `position`：基于屏幕矩形和边距的默认位置。
- `attributes`：`topmost`、`alpha`、`transparent_background`、`lock_size_across_monitors`。
- `roam`：`enabled`、`interval_seconds`、`speed`、`distance`、`multi_monitor`、`current_monitor_only`、`center_allowed`。

## 视觉方向

### 风格关键词

- 暖色桌面陪伴控制台
- 轻量 Windows 桌面软件
- 温和宠物档案
- 本地数据安全
- 可长期打开、不打扰

### 禁止方向

- 不要网页 hero、营销大标题、滚动长落地页。
- 不要大面积深色科技风、霓虹、强渐变、玻璃拟态堆叠。
- 不要做成企业后台、CRM、数据分析 dashboard。
- 不要用 emoji 当结构图标。
- 不要把纪念宠物表达成“复活”“替代真实宠物”。

## 设计 Token 建议

颜色以浅色模式为主，深色模式可作为后续扩展。

| Token | 建议值 | 用途 |
| --- | --- | --- |
| `surface.window` | `#FFF7EC` | 控制面板背景、弹窗外层 |
| `surface.card` | `#FFFDF8` | 常规卡片 |
| `surface.cardAlt` | `#FFFAF3` | 列表项、次级卡片 |
| `surface.warningSoft` | `#FFF1DF` | 危险区背景、提醒弹窗暖色区域 |
| `border.default` | `#EAD7BD` | 卡片边框、分割线 |
| `text.primary` | `#30261D` | 主文字 |
| `text.secondary` | `#7C6650` | 说明文字 |
| `brand.primary` | `#D98B2B` | 主按钮、当前选中 |
| `brand.primaryDark` | `#8A4B12` | 主按钮文字或强调 |
| `state.sage` | `#6F8F72` | 可用、已连接、已保存 |
| `state.danger` | `#C75F4A` | 删除、清除、失败 |
| `state.info` | `#B77A32` | 提醒、等待、处理中 |

字体：

- Windows 默认优先：`Microsoft YaHei UI`。
- 字号层级：页面标题 16-18pt，卡片标题 10-12pt，正文 9-10pt，辅助说明 8-9pt。
- 不使用超大标题。桌面控制台内部不需要 hero-scale 字体。

间距：

- 基础间距使用 4/8px 体系。
- 卡片内边距 12-16px。
- 卡片间距 8-12px。
- 常用按钮高度 32-36px；关键按钮点击区域接近 44px。
- chip 高度 24-28px。

圆角和边框：

- 卡片圆角 8px 以内。
- 按钮圆角 6-8px。
- 弹窗圆角可 10px，但不要卡片套卡片。
- 阴影极轻，优先用边框和背景层级。

## 信息架构

控制面板使用 920x640 作为默认尺寸，小屏允许缩窄但不能横向溢出。保留 Windows 原生标题栏，窗口名称统一为“桌宠控制面板”。

左侧导航建议分组：

- 日常：`首页`、`对话`、`提醒`、`操作`
- 宠物资产：`形象`、`档案`、`故事`、`动作`
- 智能：`AI`
- 设置与安全：`外观`、`行为`、`巡逻`、`安全`

右侧内容结构：

1. 页面标题和一句状态说明。
2. 状态总览卡。
3. 高频操作区。
4. 列表或详情区。
5. 高级设置或危险区。

每个页面只保留一个主动作。危险动作远离主动作，使用柔和红色和确认弹窗。

## 核心组件

### `PanelButton`

变体：

- `primary`：主动作，如保存、添加、测试并启用。
- `neutral`：普通动作，如编辑、查看、选择文件。
- `ghost`：低强调跳转。
- `selected`：当前选中，深蜂蜜色。
- `danger`：删除、清除、重置。
- `disabled/loading`：不可点击，文案明确如“测试中...”。

按钮文字不能竖排；窄窗口下应换行布局，而不是缩到不可读。

### `StatusPill`

用于表达：

- 当前、已配置、待配置、已保存 Key、测试中、失败、重要、逾期、今日、已加入右键、待补动作。

不能只靠颜色表达状态，必须有文字。

### `PanelCard`

用于功能块或重复项，不做卡片套卡片。卡片标题稳定，右上角最多放 1-2 个轻操作。

### `PanelSelect`

下拉需要：

- 选中项截断显示。
- 弹层根据剩余空间向上或向下展开。
- 选项多时内部滚动。
- 不使用 Windows 默认蓝色高亮破坏风格。

### `PanelSlider`

用于大小、透明度、动画速度、移动速度。结构是标题、当前值、暖色轨道、圆形旋钮。不要直接使用系统默认 `Scale` 外观。

### `PanelSwitch`

卡片式开关，适合是否自动说话、是否置顶、是否限制在屏幕、是否多屏巡游。

### `PanelToast`

用于保存成功、AI 测试成功/失败、导出结果。右上角短暂显示，不抢焦点，可手动关闭。

### `PanelDialog`

用于 Key 替换、删除、清除、导出失败、危险操作确认。必须有取消路径和 Escape 关闭。

### `ImageDropZone`

用于主像素图、现实参考照片、基础动作条、扩展动作条上传。必须同时支持拖拽和选择文件。

### `ActionPreviewCard`

展示动作名称、帧数、素材来源、右键状态、播放按钮。播放按钮只表达“预览”，不等于“加入右键”。

### `TodoCard`

展示标题、时间、分类、优先级、重复、稍后、持续提醒、当前选中态。点击选中，双击编辑。

### `ProviderStatusCard`

展示当前厂商、模型、连接方式、Key 状态、测试状态和恢复建议。

### `ChatCapabilityBar`

固定展示五类能力：云端、时间、资料、待办、本地兜底。随 AI 开关、Key 状态和待办数量刷新。

## 页面规格

### 桌面宠物窗口

- 透明背景，只显示宠物精灵和气泡。
- 支持拖动、双击摸摸、右键打开快捷面板。
- 置顶可配置，不默认强行置顶控制面板。
- 气泡短句，位置不要挡宠物脸。
- 拖动时显示跑动姿态，停止后回到待机。

### 右键快捷面板

尺寸建议：宽 280-340px，高度随内容但不撑满屏幕。

结构：

1. 当前宠物头像、名字、等级或动作包状态。
2. 常用入口：聊天、控制面板、提醒、切换形象。
3. 基础动作：挥爪、跑一小段、跳一下、等一下。
4. 扩展动作：最多显示 4 个，更多进入“更多动作 +N”弹层。
5. 窗口操作：隐藏气泡、固定/取消固定、退出。

内部按钮点击不能被失焦关闭抢断。点击外部才关闭。

### 首页总览

首屏要给用户一个“它现在怎么样”的感觉，而不是设置表单。

建议区域：

- 当前宠物大卡：头像/像素预览、名字、种类、当前动作包、切换入口。
- 陪伴状态：等级、经验条、今日陪伴、互动次数。
- 能力状态：AI、时间、资料、待办、本地兜底。
- 今日提醒：今日待办、逾期、重要事项。
- 快捷入口：和它聊、摸摸、添加提醒、去动作页、去形象页。

危险动作不放首页。

### 形象页

结构：

1. 当前主形象预览：像素主图、现实照片数量、动作包级别。
2. 主操作：切换、编辑资料、添加照片、新增宠物。
3. 家人形象列表：每只宠物一张卡，显示缩略图、状态 chip、种类、照片数、动作数、说明。
4. 操作按钮在卡片内容区内，使用稳定宽度或可换行操作行，不放窄操作列。
5. 现实照片区只展示用户上传现实照片和补充参考，不展示 spritesheet、contact sheet 或生成中间图。

新增宠物向导：

- Step 1 资料：名字、细分类、说明。
- Step 2 主像素图。
- Step 3 现实参考照片。
- Step 4 五个基础动作条：待机、向右跑、向左跑、挥爪、跳一下。
- Step 5 校验结果：尺寸、透明、帧数、动作语义。

### 对话页和聊天窗

对话页是设置和预览；聊天窗是实际交流。

聊天窗最小宽度参考 520px，高度参考 660px。底部输入区必须固定在窗口底部，历史区单独伸缩。

结构：

- 顶部：当前宠物、当前角色 Skill、能力状态条。
- 快捷工具区：两行 4 列或自适应网格，含陪伴、时间、待办、查资料、背景等入口。
- 消息区：主人和宠物气泡区分；宠物回复短句为主。
- 角色 Skill：8 个泛化风格 preset，展示当前角色说明卡和边界。
- 输入区：暖色输入卡、空输入内联提示、发送中禁用。

AI 不可用时，不显示灾难性错误；展示本地兜底和可恢复提示。

### AI Provider 页

结构：

1. 当前厂商状态卡：已连接、待配置 Key、已关闭、测试中、失败。
2. Provider 选择：OpenAI、DeepSeek、Kimi、智谱、小米 MiMo、Gemini、Qwen、OpenRouter、Mistral、Groq、自定义中转。
3. 模型选择。
4. 连接配置：Base URL、API 格式、环境变量名。
5. 本机 Key：保存、替换、清除；真实 Key 永不显示。
6. 测试结果：进行中、成功、失败和恢复建议。

Key 替换和清除必须用暖色本地确认弹窗。

### 提醒页

结构：

- 快速新增：标题、时间、重要程度、添加。
- 筛选：搜索、分类、焦点、完成状态。
- 待办列表：固定高度滚动卡片。
- 详情区：当前项、完成/重开、编辑、置顶、时间轴、稍后提醒、删除。
- 到点弹窗：标题、状态 chip、分类、优先级、提醒次数、备注预览、完成、打开提醒页、稍后、关闭。
- 时间轴：摘要卡、倒序事件卡、空状态。

“查看待办”和“新增提醒”在文案和状态上必须分清，避免用户查询时误新增。

### 动作页

结构：

1. 动作概览：基础动作数、可选扩展、已加入右键。
2. 动作预览：每个动作显示名称、帧数、素材来源、播放按钮、右键状态。
3. 右键动作列表：基础动作固定，扩展动作可加入/移出。
4. 上传区：推荐待补动作、自定义扩展动作。
5. QA 结果：尺寸、帧数、透明边缘、方向语义、是否接入右键。

动作页不能把“能播放”和“已加入右键”混成一个状态。

### 操作页

按分组展示：

- 常用操作：摸摸、说句话、召回、陪我一会。
- 基础动作：挥爪、跑一小段、跳一下、等一下。
- 扩展动作：只显示已加入右键栏的动作。
- 窗口操作：隐藏气泡、置顶、退出。

切换到操作页时滚动位置回到顶部。

### 档案页

结构：

- 宠物列表：当前宠物排第一，但查看档案不等于切换当前桌宠。
- 陪伴指标：等级、经验、陪伴时长、互动、说话、巡游。
- 长期记忆摘要：摘要更新时间、主要情绪、记忆条目。
- 最近聊天：按日折叠摘要，提供查看详情。

多宠物数据必须按宠物隔离。

### 故事页

结构：

- 宠物筛选。
- 按日期分组的故事/日记/思念列表。
- 列表项 chip：类型、时间、图片数量。
- 查看全文阅读器。
- 删除操作单独放危险区。

纪念类文案要克制。

### 外观页

结构：

- 外观总览：体型、透明度、置顶状态、气泡配色。
- 窗口行为：大小、透明度、屏幕限制、置顶、多屏尺寸锁定。
- 气泡预设：cloud、rounded、comic、minimal、pixel 等视觉卡。
- 自定义颜色：背景、描边、文字，含即时预览。

保存后显示 Toast。

### 行为和巡逻页

设置项：

- 动画速度。
- 移动速度。
- 拖动灵敏度。
- 惯性。
- 自动说话频率。
- 安静时段。
- 边缘巡游、当前屏自由、多屏巡游。

多屏策略必须清楚解释：

- 主屏仅四边。
- 当前屏自由活动。
- 多屏活动。
- 只限制自动巡游，不限制用户手动拖动。

### 安全页

结构：

- 分发安全边界：默认排除、弹窗可选、永不导出。
- 个人备份：只用于本人迁移。
- 公开安装包：新用户模板必须清空聊天、待办、提醒历史、陪伴状态、API Key 和本机路径。
- 导出状态和路径。
- 日志位置和失败恢复建议。

开机启动默认关闭，不默认创建自启动 `.lnk`。

## 响应式和窗口约束

这不是移动端响应式。设计重点是 Windows 桌面常见窗口尺寸：

- 控制面板默认：920x640。
- 控制面板窄宽验收：900px。
- 聊天窗最小：520x660。
- 右键面板：280-340px 宽。
- 弹窗：优先 420-560px 宽，内容超出时内部滚动。

文本和按钮要求：

- 不横向溢出。
- 不重叠。
- 按钮文字不能被压成竖排。
- 长说明优先换行，不强行截断；必须截断时提供查看详情。

## 交互和状态要求

- 所有异步操作有进行中、成功、失败。
- 所有危险操作有确认。
- 所有保存类操作有 Toast。
- 所有可配置状态有当前值。
- 所有列表有空状态。
- 所有失败状态给恢复建议。
- 弹窗可 Escape 关闭。
- 页面切换后滚动回顶部。
- 右键面板点击内部按钮不会被失焦提前关闭。

## 隐私和安全设计要求

- API Key 不显示真实值，只显示“已保存/未保存”。
- DPAPI 加密内容、聊天记录、待办、提醒历史、陪伴状态、本机路径不进入公开包。
- 用户源图、纪念宠物照片和故事默认是隐私资产。
- 上传、公开分享、删除前必须明确确认。
- 导出公开包只作用于导出副本，不能修改本机真实数据。

## 给设计工具的直接提示词

请为一个 Windows 桌面宠物陪伴产品生成总体 UI 设计。产品名是“蛋黄桌宠”。它不是网页、不是移动 App、不是营销页。核心界面包括透明桌宠窗口、右键快捷面板、920x640 控制面板、520x660 聊天窗、提醒到点弹窗和安全导出弹窗。

视觉方向是“暖色桌面陪伴控制台”：暖白背景、浅米卡片、蜂蜜色主按钮、鼠尾草绿状态、柔和红危险操作、轻边框、低阴影、圆角 8px 以内。使用 Windows 桌面控件思路，保留原生标题栏，不要全屏 hero、不要深色科技风、不要大面积渐变、不要 emoji 图标。

请输出这些页面的设计稿：桌面宠物气泡、右键快捷面板、控制面板首页、形象页、对话/聊天窗、AI Provider 页、提醒页、动作页、档案页、故事页、外观/行为页、安全页。每页要包含状态总览、高频操作、详细区域、空状态、错误状态和危险操作确认。

必须遵守：按钮文字不能竖排，chip 和长说明不能重叠，窄宽 900px 下列表操作仍可读，聊天窗 520px 宽度下底部输入区不能被挤出，右键面板扩展动作超过 4 个使用“更多动作 +N”，AI Key 永不显示真实值，公开导出包必须表达隐私排除边界。
