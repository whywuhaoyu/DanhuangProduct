## 核心结论

宠物类型从旧的扁平 `dog/cat/human/robot/other` 升级为「一级大类 + 二级类型 + 三级细分 + 细节描述」。分类用于资料展示、导入/编辑 UI、基础动作命名、扩展动作推荐和 AI 生图提示词，不改变现有宠物 ID、目录、历史状态和动作图集槽位。

- Tk 主程序：`data-dev/current-runtime/danhuang/run-danhuang-desktop-pet.py`
- 宠物注册表：`data-dev/current-runtime/danhuang/pet-family.json`
- 模块化核心：`src-prototype/modular/core/pet_model.py`

## 数据契约

每个宠物新增或规范化以下字段：

| 字段 | 含义 | 示例 |
| --- | --- | --- |
| `category_group` | 一级大类 | `real_animal` |
| `category` | 二级类型 | `canine` |
| `category_subtype` | 三级细分 | `long_hair_dog` |
| `category_detail` | 用户可读细节 | `奶油色松狮小狗，圆脸厚毛、粉棕鼻、灰色胸背带` |

派生信息不直接要求写入宠物数据，由分类预设提供：

- `body_profile`：体态画像
- `movement`：默认运动方式
- `base_labels`：五个基础动作槽位的显示语义
- `recommended`：推荐扩展动作

## 分类体系

一级大类：

- `real_animal`：真实动物
- `human_character`：人物角色
- `fantasy_character`：幻想生物
- `mechanical_object`：机械物件
- `plant_nature`：植物自然
- `other`：其他

二级类型与三级细分：

- 犬科：小型犬、中型犬、大型犬、短腿犬、长毛犬、短毛犬、尖耳犬、垂耳犬、狼/狐类
- 猫科：家猫、橘猫、长毛猫、短毛猫、幼猫、大型猫科
- 兔类：立耳兔、垂耳兔、侏儒兔
- 小型哺乳：仓鼠、豚鼠、龙猫、刺猬、雪貂、松鼠
- 大型哺乳：马、鹿、羊/牛、熊猫/熊、象
- 鸟类：小型飞鸟、鹦鹉、鸽类、家禽、水禽、企鹅、猛禽
- 爬行/两栖：龟、蜥蜴、蛇、守宫、蛙
- 水生：金鱼、热带鱼、水母、章鱼、海马、鲸/海豚
- 节肢：蝴蝶、蜜蜂、甲虫、蜘蛛、蟹/虾
- 人物：纪念人物、Q 版人物、像素人物、动漫角色、职业人物、儿童、老人
- 幻想：龙、精灵、史莱姆、独角兽、小怪物、幽灵、魔法宠物
- 机器人：盒子机器人、人形机器人、机甲、设备、载具
- 拟物/物件：毛绒玩具、杯子、食物、书本、工具、家具
- 植物：盆栽、花、树苗、多肉、蘑菇
- 自定义：未知/自定义

## 旧数据兼容

- 旧 `category: dog` 兼容映射到 `small_dog`，但会根据 `species` 和 `notes` 优先细化为短腿犬、长毛犬、短毛犬等。
- 旧 `category: cat` 兼容映射到 `house_cat`，如果描述包含橘猫特征则推断为 `orange_cat`。
- 旧 `category: human/robot/object` 仍可读取，但保存后会写入新的四个字段。
- 旧宠物 ID、目录名、图片路径、动作 ID、聊天记忆和状态文件不迁移、不重命名。

## UI 入口

- 形象页当前宠物卡片显示分类、细节、体态、动作摘要和基础动作语义。
- 编辑资料弹窗使用三层选择：
  - 一级大类按钮
  - 二级类型按钮
  - 三级细分按钮与搜索框
  - 摘要区显示体态、动作和推荐扩展动作
- 新增宠物弹窗复用同一分类选择器，基础动作按钮会按细分类型刷新。

## AI 与动作提示词

生图提示词会带入：

- 一级/二级/三级分类
- `category_detail`
- 体态画像
- 分类动作方向
- 推荐扩展动作
- 当前分类下的身份锁定规则

人物、鸟类、水生、机器人、物件等不再默认使用猫狗动作语义。

## 验收标准

- `pet-family.json` 可被 JSON 解析，且无 UTF-8 BOM。
- Tk 主程序和模块化核心通过 `python -m py_compile`。
- 新增宠物和编辑资料弹窗中，分类按钮不横向挤压，不出现文字竖排。
- 切换犬、猫、人物、机器人、水生等分类时，基础动作名称和 prompt 语义随分类变化。
- 保存后重启仍保留 `category_group/category/category_subtype/category_detail`。
- 现有 5 个宠物仍可切换，动作列表和历史数据路径不变。

## 风险与取舍

- 本版只增强分类、UI、提示词和数据契约，不重做 spritesheet。
- 三级细分使用内置预设，暂不开放用户自定义一级/二级枚举。
- `category_detail` 可自由填写，用来补足预设不能表达的具体品种、人物身份、材质或特殊外观。
