# 当前 Tk 桌宠 UI 优化清单

更新时间：2026-07-03

## 结论

当前版本 UI 先按“保留 Python/Tk、优化控件体系和信息架构”的路线推进，不再同步 WPF 产品版。截图中红框提到的 `CustomTkinter`、`ttkbootstrap` 只作为中期 Tk 现代化验证；`Avalonia`、`Qt for Python/PySide6`、`pywebview` 只作为未来重新选型时的备选，不覆盖当前 Tk 主线。

所有当前 Tk 版本新增 UI 功能和优化都视为产品资产，后续必须固化到 Tk 文档、运行镜像和 exe 包：

- 每次 UI 改动前，先在本文登记优化项、页面范围和验收口径。
- 每次 UI 改动后，补充截图或验收记录到 `qa/`。
- 每次 UI 改动必须检查按钮文字、状态 chip、长标题和长说明在常规窗口宽度下不重叠、不竖排、不横向溢出；不得依赖窄列硬塞操作按钮。
- 每次 UI 改动后必须同步 `data-dev/current-runtime/danhuang/`，并重新生成 Windows exe 包到 `packages/`；新包验证通过后只保留最新一组 Tk Windows 包，删除旧打包记录。
- C 盘历史运行实例已同步到 `data-dev/current-runtime/danhuang/`；后续新增功能、UI 优化和试运行优先只在 E 盘完成。

## 参考输入

- 用户截图中红框一：短期/中期可参考 `CustomTkinter` 或 `ttkbootstrap`。
- 用户截图中红框二：跨平台或 Python 重 UI 可参考 `Avalonia`、`Qt for Python/PySide6`、`pywebview`。
- 当前审计结论：`run-danhuang-desktop-pet.py` 约 15945 行，`open_control_panel` 约 3874 行，控制面板已成为主要维护风险。
- 当前代码现状：已有暖色主题、自绘 `select_box()`、自绘 `panel_button()`、滚动正文、左侧导航、右键快捷面板和多宠物缩略图。
- 当前 E 盘运行镜像：`data-dev/current-runtime/danhuang/`；代码改动需要同步 `src-prototype/legacy-monolith/run-danhuang-desktop-pet.py` 和镜像主程序。

## 技术路线判断

| 路线 | 当前判断 | 使用方式 | 验证重点 |
| --- | --- | --- | --- |
| 现有 Tk + 自绘控件 | 近期主线 | 先统一控件、导航、页面密度和弹窗，不改透明桌宠窗口链路 | 语法、JSON、右键、下拉、滚动、错误日志、截图 |
| CustomTkinter | 中期候选 | 在独立 spike 验证控制面板壳、表单、按钮和切换控件 | 透明窗口、置顶、DPI、多屏、右键层级、打包体积 |
| ttkbootstrap | 中期候选 | 只验证是否能快速改善 ttk 视觉，不作为默认风格 | 是否偏 Bootstrap、是否破坏暖色陪伴感、下拉稳定性 |
| Avalonia | 远期重新选型备选 | 不先投入正式实现 | Windows 原生贴合度、透明窗口、多屏 DPI、打包 |
| PySide6 / Qt for Python | Python 重 UI 备选 | 适合复杂面板和图形能力验证 | 打包体积、样式体系、透明桌宠窗口、资源占用 |
| pywebview | 面板复杂到需要 Web UI 时备选 | 只考虑控制面板，不直接替代宠物透明窗口 | 窗口层级、焦点、内存、离线资源、打包 |

## 分页面优化清单

### 1. 控制面板外壳

- 将窗口固定基线从单纯 `880x620` 升级为可配置尺寸策略：默认 920x640，小屏不溢出，最大化使用原生窗口状态。
- 左侧导航增加分组标题和高级折叠，低频项默认收纳：巡逻、外观、安全、生成/导出。
- 页头只保留页面标题、简短状态和置顶/最大化；全局保存、恢复默认、高级模式放到底部或设置页，避免每页都产生无关动作。
- 正文滚动保持现有机制，但页面切换后必须重置到顶部，滚动条只在需要时出现。
- 给页面加载失败的降级页保留，但补“查看日志位置”和“复制错误摘要”按钮。

### 2. 首页/陪伴

- 把“陪伴”拆成更清晰的首页：当前宠物卡、等级/经验、今日陪伴、AI 状态、提醒数量、快捷入口。
- 保留短句陪伴气质，不把首页做成后台仪表盘。
- 快捷入口最多 4-6 个：聊天、摸摸、提醒、AI、动作、切换形象。
- 危险动作如重置陪伴移到档案或安全确认弹窗，不放首屏。

### 3. 形象和宠物资产

- 已落地：当前宠物主预览放在页面顶部，展示主像素图、现实照片数量、动作包状态和主操作。
- 已落地：宠物列表拆成独立“家人形象列表”区域，行内使用状态、动作包、照片/动作数量 chip，描述在内容区换行。
- 已落地：取消形象列表窄操作列，操作按钮移入每张卡片内容区并使用固定像素宽度，避免“切换/编辑资料”被挤成竖排。
- 已落地：现实照片区只展示现实照片和补充参考，不展示 contact sheet、spritesheet 或生成中间图。
- 新增宠物导入继续保留 5 个基础动作要求，但优化为分步向导：资料 -> 主像素图 -> 参考照片 -> 基础动作 -> 校验结果。
- 已落地：新增宠物和编辑资料接入细分类模型，覆盖犬类、猫类、兔类、小型哺乳、大型哺乳、鸟类、爬行动物、水生生物、昆虫/节肢、人物、幻想生物、机器人、拟物/物件和其他。
- 已落地：基础动作槽位继续兼容旧 atlas，但 UI 和生图提示词按分类展示动作语义，例如人物是站立/走路/招手，鸟类是跳步/拍翅，水生生物是漂浮/游动，拟物角色是轻晃/弹跳。
- 已验收：`qa/tk-family-list-actions-inline-900-20260617.png`、`qa/tk-family-list-actions-inline-900-scrolled-20260617.png`，900px 宽度下形象列表按钮不竖排、不重叠。
- 图片上传区域保留拖拽和选择文件两种路径，错误提示必须落在当前区域内。

## 组件资产化要求

短期 Tk 版本需要先沉淀一套轻量 UI 组件，不继续在各页面复制 `tk.Button` 和颜色：

- `theme_tokens`：背景、卡片、文字、弱文字、主色、主色深、主色浅、鼠尾草、危险色、边框。
- `PanelButton`：primary、neutral、ghost、danger、selected、disabled。
- `PanelCard`：标题、说明、内容、危险区。
- `PanelSelect`：统一替代 `select_box()` 和 `todo_select_box()`。
- `PanelSlider`：统一替代 Tk 默认 `Scale` 和局部自绘滑杆。
- `PanelSwitch`：替代普通 Checkbutton，表达开关状态。
- `PanelDialog`：替代高频 `messagebox`，支持确认、错误、成功和异步进度。
- `StatusPill`：表达当前、已配置、待补、失败、重要、逾期等状态，不能只靠颜色。
- `ImageDropZone`：主像素图、参考照片、动作条上传统一落点。

后续 Tk 迭代时，这些组件应优先收敛为共享组件，而不是继续按页面硬写控件。

## 验收口径

每个 UI 优化批次至少完成：

- 文档：本文补充对应条目，必要时同步 `UI_REDESIGN_SPEC.md`、项目梳理和执行追踪。
- 静态验证：`python -m py_compile run-danhuang-desktop-pet.py`。
- 数据验证：settings、family、state、todos、providers JSON 可解析。
- 页面烟测：控制面板能打开目标页，切页滚动回顶部，文本不重叠，不横向溢出。
- 控件烟测：右键面板、下拉、滑杆、开关、上传入口、弹窗、危险确认可操作。
- 视觉验收：至少留下关键页截图到 `qa/`，大 UI 改动需要 contact sheet 或动作记录。
- 运行态：如果改 C 盘运行实例，验证后通过 `start-danhuang-desktop-pet.ps1` 重启当前桌宠。
